Footnotes, Endnotes, Popup Notes

Helpful to Know

To make footnotes in an ebook, it is helpful to know

  • Common terms used in book publishing
  • HTML document structure
  • Epub version you’re creating

Important to Know

Reflowable EPUBs do not have fixed pages. Because of this, reproducing traditional footnotes at the bottom of a page/view is not possible.

What is a Footnote?

Exactly what is or is not a footnote is hard to define. For our purpose here, let's use these definitions.

Popup Notes
note that popups up on screen as a result of user action, such as clicking a superscript number.
Footnotes
notes at the end of a section, like an HTML document or <div>.
Endnotes
are in a chapter or document by themselves, usually all together at the back of a book.
Annotation
is a note the user writes within the book. This is a separate topic and not covered here.
Source
Text or number in the normal flow of the book that points to a footnote's content.
Content
The text or media contained in the note itself.
Bi-directional linking
When the Source links to the Content and the Content links back to the Source

Reading System Considerations

Each reading system responds to the needs of its users in its own way. That makes for variations in the way notes are displayed. A Reading System will generally override an Ebook maker’s specifications if doing so makes the user experience better. Ebook makers do not necessarily have access to information about all the variations and overrides in each reading system.

Styling the note’s display is mostly up to the reading system.

In some cases, the reading system displays all types of notes as popups.

Testing is always a good idea.
